Lesson 16

Building Sections

Big Picture: The Next Steps of Building Out Any Web Application

First pat yourself on the back! Getting to this point in the Flixter application is a lot of work, and you're doing an awesome job! Our Flixter application now has the ability to create courses and we're well on our way to having an awesome platform built. Since courses are a core component of this web application, it's a good feeling knowing that we have a lot of that work squared away.

Before tackling specific feature sets, like video uploading, or payments, it's important to flesh out all the core data of the application.

The next core data component that we should think about building on our application is the lesson, because that seems like a critical component of the application, as well.

But first, take a step back and think about the user flow of the application. You should realize that before a user can create a lesson, they need to first build the section, the group of lessons, that lesson will eventually belong to.

This means that even though our instincts might be to build lesson functionality, we'll make faster progress by working with the next logical core of the user flow first. This means the next logical step for us to build out is the sections. In this lesson we will begin building out the sections, which will allow us to group lessons, when we build those.

Setting up sections

Since the sections will be saved in the database, it means we need to have a model for it. To generate the section model we can run the following command: 

$ rails generate model section

Like we've done in the past, after we generate a new model, the next step is to modify the migration file. The migration file will give us the initial structure for the database table. Think through the data we'll want to keep track of in the database for a section. In this case there's only a couple things we'll need to store. We'll have a title of the section, and we'll also need to keep track of the course_id that this section belongs to. So open up the migration file, XXXXX_create_sections.rb and add the title and course_id fields like this: 

class CreateSections < ActiveRecord::Migration
  def change
    create_table :sections do |t|
      t.string :title
      t.integer :course_id
      t.timestamps
    end

    add_index :sections, :course_id
  end
end

Make sure to save the file, and then run the migration: 

$ rake db:migrate

Whenever we build a new model, that links to another model in our database we need to tell Ruby on Rails how the models connect to each other. In this case the course is linked up to a section, these links are called associations. It's good to get in the habit of immediately setting up the associations after running the database migrations. By doing so we avoid strange error messages due to missing database relationships and save time debugging our web application.

Let's edit app/models/section.rb and make the section belong to a single course. We make this relationship a belongs to relationship, because the sections table of the database has a course_id, so there is only a single course any particular section can belong to. To make this database association, add this line: 

class Section < ActiveRecord::Base

  belongs_to :course
end

Save the file.

Let's add the relationship in the course model that specifies that a particular course can have many sections. This is the case in this situation because since the sections table in our database has a course_id field. This means there could be multiple rows in our sections table that have the same course_id value. Edit app/models/course.rb to include the has_many relationship: 

class Course < ActiveRecord::Base
  belongs_to :user

  has_many :sections

  validates :title, presence: true
  validates :description, presence: true
  validates :cost, presence: true, numericality: {greater_than_or_equal_to: 0}
end

Save the file.

Now that our database is properly setup and our model is configured with the proper associations, we can keep coding and add more functionality to our web application.

The next step we should take is to empower instructors to be able to create a section in the database from within our web application.

Building the form for the section on the page

If we look back at our wireframes we see that we want the form for a section to exist in a modal that pops up when a user clicks the New Section button.

Setting up the modal will require more work on the front end than having a new form on its own page. Rather than tackling both the modal, and the form at the same time, let's split the tasks.

First, let's create the form and put it on its own page, the sections new page. Later we can go back and push the form into a modal dialog.

Thinking about the user flow, we can imagine what the process will be to build a course. First an instructor will sign in to our application. Then they will create a course in the database. Once they add a new course, they will create different sections they can organize their course into, like Week 1, Week 2, etc. Ultimately they will be able to create different lessons within the sections, like Installing the Tools, Setting Up Accounts, etc.

As a first step of building the form, we're going to build up a new controller to allow us to have pages for sections. As expected, we will build a sections controller for this. However, this is a feature that is only going to exist on the instructor side of our web application, so just like we did with adding a new course, we should add the controller inside the instructor namespace. This means we're going to build the instructor/sections controller: 

$ rails generate controller instructor/sections

That command generates the controller for us, but we still need to connect the routes for it. Right now we know we're going to want to support a section new form, and a section create action for instructors.

Keep the user flow in mind; it should make sense that on both of the actions the course_id needs to be passed through the URL. The reason we need to pass the course_id through is because we need a way to know the course we are building a new section for. We can get the course_id into the URL we hook the new and create actions by "nesting" the sections underneath the course resource.

Nest the resources for sections inside the courses by editing config/routes.rb to look like this. 

Flixter::Application.routes.draw do
  devise_for :users
  root 'static_pages#index'
  resources :courses, only: [:index, :show]
  namespace :instructor do

    resources :courses, only: [:new, :create, :show] do
      resources :sections, only: [:new, :create]
    end
  end
end

Save the file.

Now if we run rake routes: 

$ rake routes

We can see that we got the two new lines added to our routes file: 

   instructor_course_sections POST   /instructor/courses/:course_id/sections(.:format)     instructor/sections#create
new_instructor_course_section GET    /instructor/courses/:course_id/sections/new(.:format) instructor/sections#new

You will notice that the URL contains :course_id, because it was nested within the courses resource in the routes file. This will allow us to extract the course_id from URL in the instructor/sections controller by running params[:course_id]. This is exactly what we want.

We need to setup the controller to have a place for the form to live. Open up app/controllers/instructor/sections_controller.rb and add the new action like this: 

class Instructor::SectionsController < ApplicationController

  def new
  end
end

Save the file.

Let's create a view for the sections new form to live. To do that, create a new file at app/views/instructor/sections/new.html.erb, and throw in a div with the standard CSS styles applied: 

<br />
<div class="booyah-box col-xs-10 col-xs-offset-1">
  <h1>Add a new section</h1>
</div>

Save the file.

Right now there is no way for instructors to get to this page we just built. To allow instructors to get to this page, we should add a link from the instructor course show page.

The first step to building a link to our application is to lookup the lines in the routes table for this particular page. So like usual, let's run the command to output the routes table: 

$ rake routes

And we see this line:

new_instructor_course_section GET    /instructor/courses/:course_id/sections/new(.:format) instructor/sections#new

Judging from this line, it means that the link will go to the new_instructor_course_section_path, and we'll need to pass through information about what course we're building a new form for.

We want to add the link to the instructor's course show page. Now that we know where our link is going, edit app/views/instructor/courses/show.html.erb and add the following link: 

<br />
<div class="booyah-box col-xs-10 col-xs-offset-1">
  <h1 class="text-center"><%= @course.title %></h1>

  <div class="text-center">
    <%= link_to 'Add a new section...', new_instructor_course_section_path(@course), class: 'btn btn-primary btn-lg' %>
  </div>
</div>

Save the file and navigate to the instructor's course show page. You'll notice the link got added.

Next, let's add a little more whitespace from the title and the Add a new section... link, by adding a br tag: 

<br />
<div class="booyah-box col-xs-10 col-xs-offset-1">
  <h1 class="text-center"><%= @course.title %></h1>

  <br />
  <div class="text-center">
    <%= link_to 'Add a new section...', new_instructor_course_section_path(@course), class: 'btn btn-primary btn-lg' %>
  </div>
</div>

Save the file and refresh the page. The spacing now looks better.

We just hooked up that a button from the instructor's course show page. If you click the button and you'll be delivered to the location where your new section form should live.

While we're editing the instructor's course show page, let's also add a button to preview the course as a student on the instructor's show page. Like always, we need to lookup the lines in the routes table for this student course show page. So like usual, let's run the command to output the routes table: 

$ rake routes

And we see this line: 

course GET    /courses/:id(.:format)                             courses#show

This means the link will go to the course_path, and we'll need to specify the course. Edit app/views/instructor/courses/show.html.erb, to add the link: 

<br />
<div class="booyah-box col-xs-10 col-xs-offset-1">
  <h1 class="text-center"><%= @course.title %></h1>
  <br />

  <%= link_to 'Preview As Student...', course_path(@course), class: 'btn btn-warning pull-right' %>
  <br class="clr" />

  <div class="text-center">
    <%= link_to 'Add a new section...', new_instructor_course_section_path(@course), class: 'btn btn-primary btn-lg' %>
  </div>
</div>

Save the file and refresh the instructor course show page. Awesome, it added the link!

Add a Form to the Instructor Sections New Page

We've hooked up much of the foundation to support sections in our application. Now is the time to flesh out the section new form so users can actually enter data.

The first thing we'll want to do is find and load the course that we're building a section for inside of our section controller. We need to get the course_id from the URL and the load up the item. Since this is passed along the URL, and the details of that are shown to us in the routes table. If you output the routes table: 

$ rake routes

You will this line: 

new_instructor_course_section GET    /instructor/courses/:course_id/sections/new(.:format) instructor/sections#new

The routes table shows us that inside the URL there is a section for :course_id. This means we can slurp the specific value that is passed through the URL from the params, by running code like params[:course_id]. Edit app/controllers/instructor/sections_controller.rb and add the following line to load the course for the form we're displaying into an instance variable called @course: 

class Instructor::SectionsController < ApplicationController
  def new

    @course = Course.find(params[:course_id])
  end
end

Save the file.

Since we're going to be using SimpleForm and it wants us to present it with a template for what we're building a form for, we're going to need to build a blank section in our controller as well. 

class Instructor::SectionsController < ApplicationController
  def new
    @course = Course.find(params[:course_id])

    @section = Section.new
  end
end

Save the file.

Next we can add the form to our view. Usually SimpleForm is smart enough to automatically guess we want to hook up the submit button of the new form to the create action. Because the section is inside the "namespace" of instructor, we need to explicitly tell it to go to the instructor/sections controller and the create action. The first step of doing that is going to be looking it up in the routes table: 

$ rake routes

And then finding the line that represents the controller and action we want the submit button to go to. In this case it is: 

instructor_course_sections POST   /instructor/courses/:course_id/sections(.:format)     instructor/sections#create

So when we build the form we need to tell it the URL should go to instructor_course_sections_path. Edit app/views/instructor/sections/new.html.erb and build the form so it goes to the correct controller by writing code like this: 

<br />
<div class="booyah-box col-xs-10 col-xs-offset-1">
  <h1>Add a new section</h1>

  <%= simple_form_for @section, url: instructor_course_sections_path(@course) do |f| %>
    <%= f.input :title %>
    <%= f.submit "Add a section" %>
  <% end %>
</div>

Save the file and navigate to the instructor section new form. Our button doesn't fit in with the rest of our application, so let's add a adjust the submit button to use styles from Twitter Bootstrap and also space it out while we're at it: 

<br />
<div class="booyah-box col-xs-10 col-xs-offset-1">
  <h1>Add a new section</h1>
  <%= simple_form_for @section, url: instructor_course_sections_path(@course) do |f| %>
    <%= f.input :title %>

    <br />
    <%= f.submit "Add a section", class: 'btn btn-primary' %>
    <br />
    <br />
  <% end %>
</div>

Save the file and refresh the page.

It took a lot of work to get this form just how we want it. It was worth it since our form is awesome!

Making the submit button actually work and create a section

The next step is to hook up the logic that happens when an instructor presses the create button on the form we just built. When this happens we want the following three things to happen:

  • Load the correct course from our database
  • Create a section in our database, connected to the course the instructor was working with
  • Redirect the user to the corresponding instructor show course page

Since one of the steps is to send the user to the instructor show course page, let's look it up in our routes table to see where we are sending the user to: 

$ rake routes

We see that the user will be sent to this page:

instructor_course GET    /instructor/courses/:id(.:format)                     instructor/courses#show

So from that we know we're going to eventually redirect_to the instructor_course_path, but before we do that we'll need to load the current course from the database. Like we did with the new page, we'll need to load the correct course from the database.

Like before, we need to get the course_id from the URL and the load up the item. Since this is passed along the URL, and the details of that are shown to us in the routes table. If you output the routes table: 

$ rake routes

You will this line: 

new_instructor_course_section GET    /instructor/courses/:course_id/sections/new(.:format) instructor/sections#new

So this means we can get the course_id from params[:course_id], and use that to load up the correct item in the database.

We should be able to suck out all the attributes from the section form field like we did when creating other items (think: Quotes in Splurty, Places in Nomster, Courses in Flixter). Edit app/controllers/instructors/sections_controller.rb, to do each of these three tasks: 

class Instructor::SectionsController < ApplicationController
  def new
    @course = Course.find(params[:course_id])
    @section = Section.new
  end


  def create
    @course = Course.find(params[:course_id])
    @section = @course.sections.create(section_params)
    redirect_to instructor_course_path(@course)
  end

  private

  def section_params
    params.require(:section).permit(:title)
  end
end

Save the file.

Test it out by navigating to the instructor's new section form. Enter a title and press the submit button. If everything works as expected you will get redirected to the instructor course show page.

Unfortunately, you won't be able to see the sections in your database displayed on your web application because we haven't added that feature yet. Let's add that step last in this lesson to make sure our item is properly being saved to the database.

Listing out the Sections of an individual course

As a last step on the instructor course show page, let's list out all the sections that exist for our course. Edit app/views/instructor/courses/show.html.erb so it has the lines of code to loop through each of the sections. We'll place the sections inside a ul tag, or an Unordered List grouping. Each of the specific items will be in its own li tag, or List Item element: 

<br />
<div class="booyah-box col-xs-10 col-xs-offset-1">
  <h1 class="text-center"><%= @course.title %></h1>
  <br />
  <%= link_to 'Preview As Student...', course_path(@course), class: 'btn btn-warning pull-right' %>
  <br class="clr" />
  
  <div class="text-center">
    <%= link_to 'Add a new section...', new_instructor_course_section_path(@course), class: 'btn btn-primary btn-lg' %>
  </div>


  <ul>
    <% @course.sections.each do |section| %>
      <li>
        <%= section.title %>
      </li>
    <% end %>
  </ul>

</div>

Save the file and navigate to the instructor course show page. Notice how there now are bullet-points listed for each of the sections we have.

This was a lot in one lesson! Awesome work following along!

How to get a fast answer for your question:

Here’s how to ask smart questions in the forum:


First, give it a good title that describes what you’re struggling with. Descriptive headlines like “my quotes don’t show on my index page“ or “the email notification email is not triggered at comment submission” work best.


Second, describe in detail what you’ve done up to this point. For example, you should say things like: “I added a quote through the rails console and can see it in my database. I also added the code to create the loop on my index.html.erb page and added the necessary code inside of my quotes controller too”.


Third, describe in detail what you expected to happen. For example, you should describe things like this: I refreshed my browser window with the url “localhost:3030” and expected a new quote to be shown on my root page.


Fourth, paste a screenshot of your error message. This one is obvious, if we can see what you’re seeing we can help you a lot better. Make sure you get the whole page.


Fifth, include a link to your github repository: If we can’t figure out how to help you through your description above, we often times go directly into GitHub and take a look at your web application code, pull it onto our machines, and give it a little test drive.

Important: Make sure you always commit and push the latest version of your code to your GitHub repository, before submitting your question in the Q&A forum. By doing so we can look at your latest code and see the same error message as you.


Sixth, include a link to you web application: We might need to have a look at your app live on the internet. Including the url for your app will help. Plus if you don't include the url, you might get a response from Jarvis, the firehose robot, requesting that you add it.


Speed of help: We generally check out forum several times each day and answer all questions. The better you formulate your problem, the faster you get your solution.

This makes sense. If you forget to include a screenshot of your error message, the most likely reply that you get back will be something like “hey there, can you please include a screenshot of your error message so we can see what you’re seeing?”.


Directly emailing your Mentor a question: The forum is a lot faster way to get help - since we always have somebody on dedicated forum watch. It might seem tempting to go around the forum and email your mentor directly, but that will most likely get you a delayed response, since your mentor might be helping other students and will not get back to you as fast.
